IRS Connection
TDS API — pull client transcripts directly from the IRS
How this works: Using Romeo's IRS e-Services API credentials (A2A OAuth flow), this connects to the IRS Transcript Delivery System. The system generates two signed JWTs using Romeo's private key, exchanges them for an access token, then requests the client's transcript via CAF authority — exactly as the IRS API documentation specifies. Credentials must be set in the Credentials tab first.

IRS API Credentials
Romeo's IRS e-Services credentials for transcript access
All other credentials (Resend, Stripe, Netlify) are already configured as environment variables. This page is only for the IRS API credentials needed to pull client transcripts directly from the IRS.
🏛️ IRS e-Services API (TDS)
Romeo's IRS API credentials for the Transcript Delivery System. Required to pull client transcripts via the A2A OAuth flow. Once set, go to the IRS Connection tab to test.
IRS API Client ID
e-Services Username (used as JWT sub claim)
CAF Number (9 digits, no hyphens or spaces)
JWK Key ID (kid — from your JWK registration)
Private Key — PEM format (RSA-2048)
How to get these credentials: 1. Client ID — Log in to irs.gov/e-services → Access API Client ID → Complete application. Upload your JWK public key file. Client ID appears on Application Details page.
2. Private Key — Generate RSA-2048 key pair. Upload the public key (as JWK with x5c/x5t) to IRS during registration. Paste the private key here.
3. CAF Number — Call IRS Practitioner Priority Service at 866-860-4259 to retrieve it.
4. Grant A2A Consent — One-time step: log in to la.www4.irs.gov/esrv/consent/ and authorize your Client ID for A2A access.
